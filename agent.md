# Trip Agent Guide

本文件约定 `/Users/town/Documents/trip` 项目中，Codex/Agent 规划云南两个月背包旅行时使用的工具、MCP、数据源和执行边界。

## 基本约定

- 默认使用简体中文回答。
- 代码、命令、路径、接口字段、报错原文保持原样，不翻译。
- 本地命令默认加 `rtk` 前缀，例如 `rtk git status`、`rtk uv run ...`。
- Python 环境使用 `pyenv` 和 `uv` 管理。
- 默认旅行设定：`12000 CNY` 总预算、约 `60 天`、背包客方式、低成本优先。
- 默认预算策略：先保安全和机动金，再压低住宿、餐饮、交通和门票支出。
- 当前目录以途中使用为主：先看 `00-dashboard/`，再进入当前地点的 `03-stages/<stage>/`。
- 所有价格、余票、天气、景区开放状态都视为实时信息；做正式建议前必须重新查询或注明查询日期。
- 涉及登录、实名、支付、验证码、抢票、退改签的动作，Agent 只辅助查询和整理，不自动执行。

## 项目目录工作流

- `00-dashboard/`：当前状态、预算余额、待办、紧急信息。
- `01-gear/`：装备、采购、重量检查。
- `02-route/`：总路线和跨站备选。
- `03-stages/`：每一站的执行目录，是主要工作区。
- `04-bookings/`：只放人工确认后的订单信息。
- `05-research/`：通用工具、MCP、天气源、历史查价。
- `06-exports/`：导出版本。

每个 `03-stages/<stage>/` 统一维护：

- `plan.md`
- `transport.md`
- `stay.md`
- `food.md`
- `budget.csv`
- `risk.md`
- `backup.md`

## 推荐 MCP / API

### 地图、POI、路线

优先级：

1. 高德地图 MCP / Web Service API
2. 百度地图 MCP / API

用途：

- 查询景点、车站、机场、酒店位置。
- 估算城市间和城市内交通时间。
- 查询公交、驾车、步行路线。
- 校验住宿周边便利度。

参考：

- `@amap/amap-maps-mcp-server`
- `uvx amap-mcp-server`
- https://github.com/sugarforever/amap-mcp-server
- https://github.com/baidu-maps/mcp

### 火车票

优先级：

1. `12306 MCP`
2. 铁路 `12306` 官网/App 人工确认

用途：

- 查询车次、余票、经停站、中转方案。
- 比较 `昆明 -> 大理 -> 丽江 -> 香格里拉` 等铁路段。

边界：

- Agent 可以查询和推荐车次。
- 购票、改签、退票、候补、支付必须人工完成。

参考：

- https://github.com/Joooook/12306-mcp
- https://www.12306.cn/

### 机票

优先级：

1. 飞猪 / FlyAI CLI
2. 航旅纵横
3. 携程
4. 同程

用途：

- 查询出发城市到 `昆明`、`丽江`、`西双版纳`、`腾冲/保山` 等机场的价格。
- 比较直飞、中转、行李额、退改签规则。

边界：

- 国内 OTA 对个人自动化不友好，公开 API 多偏合作方。
- Agent 只整理搜索策略、比价表和候选航班。
- 下单、支付、值机、退改签必须人工完成。

### 住宿

优先级：

1. 飞猪 / FlyAI CLI
2. 携程
3. 美团
4. Booking

用途：

- 查询城市分段住宿价格。
- 对比可取消政策、位置、评分、交通便利度。
- 优先查询青旅床位、多人间、低价客栈、连住优惠和可做饭住宿。
- 建议住宿区域，例如大理古城/喜洲、丽江白沙/束河、香格里拉独克宗附近。

边界：

- Agent 不自动登录、不自动下单、不自动支付。
- 对雨季山区目的地优先选择可取消订单。
- 单晚住宿优先控制在 `35-80 CNY`，热门城市旺季可临时放宽，但必须在预算表里补偿。

本地 FlyAI 工具：

- 说明文档：`06-research/tools/flyai.md`
- Skill 仓库：`06-research/tools/flyai-skill`
- CLI 路径：`06-research/tools/flyai-cli/node_modules/.bin/flyai`
- API Key 从项目 `.env` 读取，变量名为 `FLYAI_API_KEY`；`.env` 已被 `.gitignore` 忽略，禁止把 key 写进文档、提交记录或对话输出。

常用命令：

```bash
set -a; source .env; set +a
06-research/tools/flyai-cli/node_modules/.bin/flyai search-hotel \
  --dest-name "昆明" \
  --key-words "青旅" \
  --check-in-date 2026-06-01 \
  --check-out-date 2026-06-02 \
  --max-price 120 \
  --sort price_asc
```

查询结果写入 `03-stages/<stage>/stay.md` 时，保留：查询时间、平台 `飞猪/FlyAI`、名称、区域/近 POI、价格、链接、取消政策待人工确认。

### 天气和预警

优先级：

1. 和风天气 API / MCP
2. 彩云天气 API / MCP

用途：

- 查询未来天气、降雨、温度、风力。

- 决定是否保留虎跳峡、雨崩、怒江、独龙江、丙察察等高风险路线。

参考：

- https://dev.qweather.com/docs/api/

## 查询工作流

### 初版规划

1. 明确出发城市、出发日期、预算、人数、行李、是否接受高海拔和徒步。
2. 根据季节和雨季风险确定路线大框架。
3. 用地图 MCP 校验城市顺序和交通耗时。
4. 用 `12306 MCP` 查询铁路主干段。
5. 用 OTA 人工比价机票和住宿，Agent 整理候选表。
6. 检查装备缺口和采购预算。
7. 输出 `行程表`、`预算表`、`装备表`、`待确认清单`。

### 滚动更新

每次临近出行前按以下顺序更新：

1. 天气和预警。
2. 道路、景区开放状态。
3. 火车余票和机票价格。
4. 住宿价格和可取消政策。
5. 当日行程是否需要替换。

### 价格记录

每条价格记录必须包含：

- 查询日期和时间。
- 平台名称。
- 出发地和目的地。
- 日期。
- 价格。
- 是否含税费、行李、早餐、接送等。
- 退改/取消规则。
- 链接或截图来源说明。

## 云南两个月旅行默认路线

默认使用昆明进出、顺时针大环线：

`昆明 -> 大理 -> 沙溪/剑川 -> 丽江 -> 泸沽湖 -> 香格里拉 -> 德钦/梅里 -> 腾冲/芒市 -> 普洱/景迈山 -> 西双版纳 -> 建水/元阳/蒙自/普者黑 -> 昆明`

可按天气和身体状态调整：

- 雨季优先减少峡谷、山路、徒步。
- 怒江、独龙江、丙察察、雨崩只作为天气稳定时的可选项。
- 香格里拉、德钦、梅里段必须保留高海拔适应和机动天。
- `12000 CNY` 背包客预算下，优先减少高价门票、索道、包车和频繁跨区移动。
- 如果预算压力过大，优先删减 `泸沽湖`、`德钦/梅里`、`腾冲/芒市` 中成本较高或交通较绕的段落。

## 安全边界

- 不安排未开放区域、野线徒步、无资质包车、非正规边境活动。
- 雨天不强行走峡谷、临崖山路、涉水路线。
- 不在山区赶夜路。
- 不把 `日照金山`、雪山索道、徒步穿越设为刚性目标。
- 出现明显高反症状时，不继续上行。
- 不为了省钱选择明显不安全住宿、黑车、夜间山路或无保障徒步。
