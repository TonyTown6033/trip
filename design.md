# 信息收集系统设计文档

## 目标

分阶段完成 10 个城市的信息搜集、分析和计划生成，使出发前（2026-05-20）尽可能多的城市有可执行的计划。

## 阶段划分

### Phase 1：天气 + 口碑数据驱动路线设计（2 天）

**输入**：当前总路线（`02-route/master-route.md`）

**任务**：
1. 用和风天气 API 查询各城市未来 7 天实时预报 + 5-7 月历史同期气候数据（降雨天数、气温范围）
2. 标记高风险时段（暴雨、地质灾害高发期）
3. 小红书搜索各城市 5-7 月旅行关键词（如"大理六月"、"香格里拉雨季"），获取：
   - 当季推荐/避坑信息
   - 各城市热度和拥挤程度
   - 时令性体验（如景迈山采茶季、泸沽湖花期）
4. 综合天气风险和口碑数据，校验路线顺序是否需要调整

**输出**：
- `05-research/weather-reference.md`：各城市气候参考
- `05-research/route-research-xiaohongshu.md`：小红书路线参考数据
- 如需调整路线，更新 `02-route/master-route.md`

**工具**：和风天气 API、小红书爬虫（MediaCrawler / 人工搜索）

---

### Phase 2：逐城市信息搜集（3-4 天）

**输入**：确定后的路线城市列表

**任务**：按优先级逐站搜集，每站产出标准化数据：

| 信息类型 | 工具 | 输出位置 |
|----------|------|----------|
| 住宿（青旅/客栈） | FlyAI CLI | `03-stages/<stage>/stay.md` |
| 景点/POI | 高德地图 API | `07-attractions/<city>.csv` → 精选入 `03-stages/<stage>/plan.md` |
| 美食/餐饮 | 高德 POI + 人工补充 | `food/food.csv` → 精选入 `03-stages/<stage>/food.md` |
| 城际交通 | 12306 MCP + 高德路线 | `03-stages/<stage>/transport.md` |

餐饮候选总表放在根目录 `food/`，与 `04-bookings/` 一样作为一级资料目录，便于 Web 直接索引和展示；各站点 `food.md` 保留执行期精选结果。

**执行顺序**（按出发时间优先）：
1. 昆明 ✅ 已完成
2. 大理
3. 沙溪/剑川
4. 丽江
5. 泸沽湖
6. 香格里拉/德钦
7. 腾冲/芒市
8. 普洱/景迈山/西双版纳
9. 建水/元阳/蒙自/普者黑
10. 昆明返程

**每站标准流程**：
```
FlyAI search-hotel --dest-name <城市> --check-in-date <日期> --max-price 80 --sort price_asc
→ 筛选 top 5 写入 stay.md

高德 POI 搜索：景点/美食/交通枢纽
→ 写入 plan.md / food.md

12306 查询前后段车次
→ 写入 transport.md
```

---

### Phase 3：多维度分析（2 天）

**输入**：Phase 2 产出的候选数据

**任务**：
1. **价格分析**：对比各平台住宿价格，标记最优选
2. **位置分析**：用高德计算住宿到主要景点/车站的距离和耗时
3. **口碑补充**（人工）：出发前/途中用小红书搜索关键词，补充评价到对应文件
4. **风险标注**：标记取消政策不明、位置偏远、评价过少的选项

**输出**：每站 `stay.md` 增加推荐排序和决策依据（参考昆明站已有格式）

**工具**：FlyAI CLI（比价）、高德地图 API（距离）、人工小红书

---

### Phase 4：生成城市计划（2 天）

**输入**：Phase 2-3 的搜集和分析结果

**任务**：为每站生成具体日程表，包含：
- **居住计划**：推荐住宿 + 备选，入住/退房时间
- **游玩计划**：每日行程安排（上午/下午/晚上）
- **交通计划**：到达和离开方式、城内交通

**输出**：更新每站 `plan.md`（参考昆明站 D1-D4 格式）

---

### Phase 5：串联与校验（1 天）

**输入**：所有城市计划

**任务**：
1. 检查相邻城市交通衔接（离开时间 vs 到达时间）
2. 汇总预算，校验是否超出 12000 CNY
3. 标记需要人工确认的节点（购票、预订）
4. 更新 `00-dashboard/overview.md` 和 `todo.md`
5. Web 前端验证所有页面数据正常展示

**输出**：
- 完整可执行行程
- 待人工确认清单

---

## 时间线

| 阶段 | 预计耗时 | 截止 | 依赖 |
|------|----------|------|------|
| Phase 1 | 2 天 | 05-16 | 和风天气 API + 小红书 |
| Phase 2 | 3-4 天 | 05-18 | FlyAI + 高德 + 12306 |
| Phase 3 | 2 天 | 05-19 | Phase 2 完成 |
| Phase 4 | 2 天 | 05-19 | Phase 3 完成（可与 Phase 3 交叉） |
| Phase 5 | 1 天 | 05-20 | 全部完成 |

Phase 3 和 Phase 4 可以交叉进行：完成一个城市的分析后立即生成该城市计划，不必等所有城市分析完毕。

| `03-stages/<stage>/food.md` | 精选餐饮 | Phase 2-3 |

## 约束

- 出发日 2026-05-20，Phase 2 优先覆盖前 4 站（昆明→大理→沙溪→丽江），后段可途中补充
- 小红书口碑为人工操作，不阻塞自动化流程
- 所有价格数据标注查询日期，出发前 1-2 天对前 2 站做最终确认
- 预订/购票始终由人工完成
